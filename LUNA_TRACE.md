# luna_trace

[Текст диплома, в рамках которого создавалась luna_trace](https://docs.google.com/document/d/1LNq2kFpozgDadfpjo_DveyH90nPe3cjkkrOU7q7lOUI/edit?usp=sharing)

## Назначение

_luna_trace_ - автоматизированное средство обнаружения семантических ошибок в
LuNA-программах.

Обнаруживаемые классы ошибок:
- Зависания, вызванные использованием неинициализированных данных.
- Повторная инициализация данных.

## Использование

### Сборка LuNA-программы

Для использования _luna_trace_ требуется информация, генерируемая при 
сборке LuNA-программы. Для этого необходимо собирать программу 
с использованием ключей `--no-cleanup` и `--build-dir=build`. 

_luna_trace_ анализирует логи, генерируемые исполнительной системой во
время работы. Генерация логов происходит только при использовании отладочной версии
исполнительной системы, поэтому при сборке также необходимо использовать ключ `-g`.

Таким образом, команда для сборки программы должна иметь вид:
```bash
luna -g --no-cleanup --build-dir=build <LUNA SOURCES>
```

### Анализ с помощью luna_trace

После завершения выполнения LuNA-программы в рабочей директории
будут сгенерированы несколько файлов с именами вида `id.*.map.list`, 
`pending.*.json.list` и `cf.*.json.list`. При запуске _luna_trace_
по умолчанию будет производить поиск этих файлов в текущей директории,
однако можно указать их явно.

_luna_trace_ также имеет следующие опциональные параметры:
- `--all` - выводить информацию обо всех зависших ФВ. По умолчанию выводится только 
информация о тех зависших ФВ, которые не имеют зависимостей по данным от других 
зависших ФВ.
- `--no-double-init` - пропустить поиск случаев повторной инициализации. Поиск может 
занимать много времени. По умолчанию поиск производится.
